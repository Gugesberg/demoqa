Модификаторы доступа:

1. private
Видно только внутри того же класса.
Не доступно ни из других классов того же пакета, ни из подклассов.
Чаще всего используют для полей и вспомогательных методов, чтобы инкапсулировать внутреннюю реализацию.

2. package-private (по умолчанию)
Уровень доступа, когда модификатор вообще не указан.
Видно внутри того же пакета, но не видно из других пакетов.
Используется, когда нужно дать доступ “внутри модуля”, но скрыть от внешнего кода.

3. protected
Доступно в том же пакете, как и package-private.
Плюс доступно во всех подклассах, даже если подкласс в другом пакете.
Часто используют, чтобы открыть часть API для наследников, но скрыть от “постороннего” кода.

4. public
Максимально открытый уровень: видно из любого класса в любом пакете.
Обычно делают public классы библиотек, сервисы и методы, которые являются частью внешнего API.

Enum - по сути тоже класс, специальный ссылочный тип (перечисление), который задаёт фиксированный набор
константных значений и тем самым ограничивает возможные варианты для переменной. Например дни недели
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

Инкапсуляция — это принцип ООП, при котором данные объекта и работающие с ними методы объединяются
в единое целое и скрываются за чётко определённым интерфейсом.
Суть инкапсуляции.
Объединяет состояние (поля) и поведение (методы), чтобы работать с
ними как с одним объектом, а не с разрозненными данными.
Скрывает внутреннюю реализацию (как именно хранятся и обрабатываются данные) и ограничивает
прямой доступ к ним извне, оставляя только “официальные” способы взаимодействия.
